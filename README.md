### Arsenal do Crime — Seized Firearms in Southeast Brazil, 2018–2023

**Instituto Sou da Paz · Public safety · Firearms policy · Data automation · Published report**

This case study documents my contribution to the data preparation stage of the Instituto Sou da Paz report *Arsenal do Crime: Análise do perfil das armas de fogo apreendidas no Sudeste, 2018–2023*.

My work involved cleaning and harmonizing data, structuring analytical variables, and extending a Python automation script to infer firearm manufacturing year from serial-number patterns for **Boito** and **CBC** firearms, building on prior work developed for Taurus-manufactured firearms.

**Methods and tools:**
Python · data cleaning · data harmonization · analytical variable construction · automation · documentation

**Why it matters:**
The project supported evidence generation on the profile, temporality, and circulation dynamics of seized firearms in Southeast Brazil, contributing to public debate and policy recommendations on illicit firearms markets.

**Key skills demonstrated:**

* Working with real-world government data
* Cleaning and harmonizing complex datasets
* Structuring analytical variables
* Automating classification rules
* Improving traceability and scalability
* Supporting a published NGO research report

**Responsible data note:**
This repository does not include the original dataset or operationally sensitive rules. It documents the analytical workflow, data-quality challenges, and reproducibility principles applied in the project.

### Firearms Import Control Cross-Referencing and Matching System

**Public policy · Government data · Record linkage · Data quality**

This volunteer project was developed in collaboration with a data team supporting a UK-based NGO operating in conflict-affected contexts. It focuses on cross-referencing and matching heterogeneous firearms import datasets to identify patterns and connections across documentation and importation data.

The project involved extensive data cleaning, transformation, harmonization, and matching processes to improve the accuracy and reliability of cross-referenced data from different sources.

**Methods and tools:**
Python · pandas · regular expressions · record linkage · data cleaning · data harmonization · reproducible workflows

**Why it matters:**
Government and administrative datasets are often fragmented, inconsistent, and difficult to compare. This project demonstrates how structured data workflows can support transparency, accountability, and evidence generation in public policy contexts.

**Key skills demonstrated:**

* Cleaning and harmonizing heterogeneous datasets
* Matching records across different sources
* Building reproducible analytical routines
* Documenting assumptions and limitations
* Working responsibly with sensitive policy data
